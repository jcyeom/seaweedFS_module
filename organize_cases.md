# organize_cases.py 사용 설명서

## 개요

tar.gz 아카이브 파일들을 카테고리별로 분류하여 압축 해제하는 전처리 스크립트.

tar.gz 내부의 중첩 디렉토리 구조를 무시하고, **엔드포인트 파일만 추출**하여 플랫하게 한 폴더에 저장한다.
SeaweedFS에 업로드하기 전 데이터를 정리하는 용도로 사용한다.

## 실행 방법

```bash
python3 organize_cases.py
```

## 동작 흐름

1. 원본 폴더에서 `*.tar.gz` 파일 탐색
2. 파일명에서 카테고리, 분류 라벨, 기간 파싱
3. tar.gz 내부의 모든 파일을 순회
4. 각 파일을 `{카테고리}_{원본파일명}` 형태로 분류 폴더에 저장
5. 파일명 충돌 시 `{숫자}_{카테고리}_{원본파일명}`으로 저장
6. 각 폴더에 `file_list.txt` 생성 (카테고리별 상세 목록)

## 원본 tar.gz 파일명 패턴

```
{카테고리}_{분류라벨}_{시작일-종료일}.tar.gz
```

## 저장 파일명 규칙

### 기본 형식

```
{카테고리}_{원본파일명}
```

### 파일명 충돌 시

서로 다른 하위 경로에 동일한 파일명이 존재할 경우, 앞에 숫자 접두어를 붙여 구분한다.

```
1_{카테고리}_{원본파일명}
2_{카테고리}_{원본파일명}
```

## 출력 구조

```
output/
├── organize_cases.py        # 실행 스크립트
├── category_a/              # 분류 A
│   ├── file_list.txt        # 파일 목록
│   └── *.jpg
└── category_b/              # 분류 B
    ├── file_list.txt
    └── *.jpg
```

## file_list.txt 구조 예시

```
=== 파일 목록 ===
=== 총 카테고리 수: 10 ===
=== 총 파일 수: 50000 ===

======================================================================
[CATEGORY_01]
  원본 tar: CATEGORY_01_label_YYYYMMDD-YYYYMMDD.tar.gz
  기간: YYYYMMDD-YYYYMMDD
  파일 수: 5000
======================================================================
  CATEGORY_01_file_001.jpg
  CATEGORY_01_file_002.jpg
  ...
```

## 비고

- 이 스크립트는 SeaweedFS 업로드 전 데이터 전처리용으로, SeaweedFS와 직접 통신하지 않는다.
- 정리된 파일은 `seaweed_upload_to_seaweedfs.py`로 SeaweedFS에 업로드할 수 있다.
